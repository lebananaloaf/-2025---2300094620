## 课程总结
我差不多两年没怎么写代码了。上一次比较认真写还是大一的《计算概论》，而且当时学的是 C 语言。后来就基本没碰编程了，所以这次回到数算，对我来说其实跨度挺大的：既要把以前学过的一些基础重新捡起来，又要适应 Python 这种新的写法和习惯。

一开始我真的有点慌。我以为会很折磨：各种 bug、各种卡壳，做题也会做得很慢。但慢慢跟着课程走下来以后，我发现情况没我想得那么糟，甚至有时候还挺有意思的。尤其是做题的时候，从一开始没思路到突然想通、再把它写出来，那种感觉其实蛮爽的。而且我也发现算法题不只是“套模板”，很多时候需要自己想办法、换角度、调整数据结构，挺锻炼人的。

不过我也很清楚自己哪里做得不够。最明显的就是：练习题做得还是太少了。很多时候我不是不会算法，而是对 Python 不够熟，写起来不顺、容易写错、调试也慢，这个对机考影响挺大。现在回头看，如果我早点多刷点题、把 Python 写熟一点，考试应该会稳很多。

但总体来说，这门课让我重新找回了一点写代码的状态，也让我对 Python 变得更有好感。我之后会继续学 Python，也想做一些自己的小项目，不想让这次只是“为了考试临时学一下”。

期末笔试那部分我一开始也挺怕的，主要是很多内容我以前没接触过，感觉概念很新、很抽象。但做了练习、认真看过之后，我反而觉得这些东西挺有意思的，尤其是 AI 背后的一些原理，让我对计算机和 AI 不是只停留在“会用”，而是开始理解它大概怎么运作。虽然我理解得还不算特别扎实，但确实开了眼界，也让我更想继续学下去。

总之，这门课对我来说更像是一次“重新回到编程”的过程：前面很生疏、很担心，但后来慢慢进入状态，也看到了自己要补的短板。之后我会把练习量补上，把 Python 写熟一点，继续往下走。
